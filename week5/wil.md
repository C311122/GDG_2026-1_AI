# 5주차 wil

## 결정트리

- 스무고개처럼 데이터를 학습
- 사이킷런의 DecisionTreeClassifier 를 통해 구현
- 과대적합 발생 시 가지치기로 방지
- Grid Search, Random Search로 가지치기 할 정도 즉 최대 깊이를 정함

## 검증 세트

- 지금까지 데이터를 train / test로 나눈 것에는 문제점 존재
- test로 과적합 방지를 인지하고 조정한 모델로 다시 같은 test로 성능확인
- 문제집 / 모고 / 수능의 모고에 해당하는 부분 필요
- 검증(validation) 데이터셋 추가

## 교차검증

- 데이터를 한 번만 train/validation으로 나누지 않고, 
여러 번 나눠서 검증하는 방법
- n번 나눈다 -> n-fold 교차 검증

## 앙상블

- 결정 트리 하나에 의존하여 생기는 문제 방지 
- 여러개의 모델을 함께 사용

## 랜덤 포레스트

- 무작위로 여러개의 결정 트리 사용
- 그 트리들을 만들 때 데이터를 랜덤하게 뽑는다: Bootstrap, 특성도 랜덤하게 고른다

## 느낀 점

로지스틱 회귀에서 만족스러운 결과를 얻을 수 없던 걸 결정 트리를 이용하여 해결하면서 그 성능을 체감하였고, 이를 다양하게 여러 개 만들어서 랜덤으로 뽑아 학습해 의존성을 크게 줄이는 방법도 신선하였다.
항상 데이터를 둘로만 분해하였는데 최종 테스트 전 모델 조정용으로 별개의 테스트 데이터셋을 하나 만드는 방법과 그 성능은 놀라웠으나 n-fold로 여러번 우려먹어도 학습 성능의 저하는 피할수 없지 않을까 생각하며 데이터의 양의 중요성에 대해 다시 한번 생각해보았다.
